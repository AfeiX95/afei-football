# 阿飞·赛博聊球 — 内容生产工作流

## 一、启动

### 理解推荐机制（每个写稿人都该知道——三阶段漏斗）

头条推荐不是"写得好就推得多"，而是分阶段的筛选漏斗。每篇文章的命运由三关决定：

```
第1关·冷启动（发布后0-1小时）
  初始流量几百人 → 看标题点击率
  目标：点击率>8%（我们已做到11%+）
  关卡控制点：标题 + 封面

第2关·小流量测试（发布后1-4小时）
  流量放大到几千人 → 看完播率+互动率
  目标：互动率≥0.2%（我们当前≈0% → 最大瓶颈）
  关卡控制点：正文质量 + Step 5互动执行

第3关·放大期（发布后4小时+）
  流量到几万甚至爆款 → 看互动持续性+关注率
  目标：持续互动+粉丝转化
  关卡控制点：评论区活跃度 + 账号垂直度
```

**核心认知：** 每篇文章都在帮算法训练你对读者的画像。持续发足球内容=算法给你的标签越来越精准（"足球创作者"），算法会把更多足球兴趣用户推给你。而断更不仅损失当期流量，还中断了标签积累——恢复需要至少2天。

**所以：** 标题定第1关的生死，互动定第2关的生死。我们的文章内容质量（第2关的完播率）已经达标，唯一的短板是互动。


用户提供：**赛事名称 + 中国竞彩网官方数据**

示例：周二007 欧冠 05-06 03:00 阿森纳VS马竞

**产出类型判断**（写稿前先定，记下上一篇是什么类型，必须轮换）：

3. 记下最近一篇的类型，写下一篇前先确认

### 三层轮换机制总览

从这篇文章开始，轮换分为三个独立维度，写稿前逐层确认：

```
第1层：风格库 → 定整体骨架（叙事流/数据先导式/对决体等7种，不连用同一种）
第2层：写作池 → 定模块组合（8大模块各选1种，至少换2个模块）
第3层：呈现手法 → 定每个模块怎么写（8种信息块各选1种手法，不连用同一种手法）
```

每篇文章的差异化 = 风格 × 模块组合 × 呈现手法的排列组合，理论上数百种变化。

## 二、文章生成流程

### Step 1：数据核查（核心步骤 — 反复犯过错，必须重视）

此步骤分两层核查，缺一不可。

**第1层：联网搜索核实基础事实**
- 用WebSearch确认：比赛性质、时间、场地
- 用WebSearch核实：**关键球员是否还在该队**（姆巴佩2024已去皇马，2026年不可能在PSG；克罗斯已退役或离队）
- 用WebSearch交叉验证：首回合比分、进球者、关键数据
- 用WebSearch核实：历史纪录类表述（"最快进球""最多连胜"必须有出处）
- ⚠️ **用户提供的文本数据可能有误**，自己的训练知识也可能过时。必须用联网搜索做二次确认

**数据铁律（2026-05-21新增）：**
- ⛔ **绝对禁止编造任何数据** — 包括但不限于：历史夺冠年份、球员转会信息、外媒/分析师观点、历史纪录、"最快进球""最多连胜"类表述
- ⛔ **外媒/分析师观点必须有真实来源** — 不能自己编造"XX外媒认为"，只能引用竞彩网数据中明确提供的信息，或用"有观点认为"模糊处理
- ⛔ **历史数据必须可查证** — 所有非竞彩网提供的历史类信息（"上次夺冠是X年""X年未赢过"等），必须用联网搜索确认后再写入
- ✅ **不确定的数据宁可不写** — 任何数据如果没有在竞彩网数据中明确出现，或者无法通过联网搜索证实，一律不写进文章
- ✅ **每次写稿前先过这条铁律** — 写完正文后逐句检查：这句话里的数字/事实是从哪里来的？来源是否可靠？如果是自己编的——删掉

**第2层：对照竞彩网截图逐条核对**

拿到用户竞彩网截图后，逐一比对：

| 核查项 | 最容易踩的坑 |
|--------|-------------|
| 积分榜排名/积分 | 新赛制（点球胜2分/点球负1分）与常规积分算法不同 |
| 联赛总战绩 | 场次/胜/平/负必须和积分榜完全一致 |
| **主客场分开战绩** | ⚠️ 客场战绩最容易被忽略，之前犯过错（神户客场写成2胜2平2负，实际3胜0平3负） |
| 近期赛果 | 注意区分联赛/杯赛/亚冠，近况描述要准确 |
| 历史交锋 | 最近5次交锋比分和趋势，注意主客场 |
| 射手数据 | 进球/助攻数字 |
| 伤停信息 | 伤停名单是否影响核心阵容 |

**核查红线（来自历史教训）：**
- ❌ 淘汰赛对手 ≠ 联赛阶段对手（最易混淆）
- ❌ 不能编造球员纪录（"最快""最多"类必须有来源）
- ❌ 不能写未经数据提供的联赛表格的数字
- ❌ **不能照单全收用户提供的文本数据** — 必须用联网搜索 + 截图交叉验证
- ❌ **封面/摘要里的数据必须精确** — "主场5战全胜"有歧义就不行，要写"主场5胜2负"
- ⚠️ **所有数字写进文章前，再读一遍原文对照** — 写稿过程中可能抄错行或看串列

### Step 2：写稿

**第一步：写标题（必过标题自检）**

标题格式：`日期 队名VS队名 钩子`（30字以内）

**核心铁律：钩子必须留悬念，不能给出答案。**

报告验证数据：
- ✅ "瓦萨客场5年零封拉赫蒂" → **18.47% CTR**（数据给了，但答案在正文里）
- ✅ "主场10连胜对7人伤缺" → **15.34% CTR**（冲突给了，谁赢没说）
- ❌ "拜仁117球收官！凯恩33球追金靴" → **1.54% CTR**（看完标题就知道了，不用点）
- ❌ "皇马刚输巴萨又伤9人" → **0.68% CTR**（新闻式陈述，没有新信息收益）

**标题自检三步：**
1. 读者看到这标题，脑子里还有没有疑问？→ 没有就是剧透
2. 删掉后面的钩子部分，只看队名+数据，好奇心还在吗？→ 不在了就重写
3. 跟上一篇是同一个句式吗？→ 轮换标题公式（翻写作池·标题手法8种轮换）

**第二步：选模块 + 选呈现手法（翻写作池 · 三层轮换）**

打开`写作池_三源融合.md`，按以下五步完成轮换：

**模块层（第2层）：**
① 查追踪表确定禁用模块（上次用了什么，这8项里哪几项不能用）
② 按比赛场景查"排列组合速查表"选推荐模块（争冠关键战/强弱分明/强强对话/中下游混战/快速出稿）
③ 从推荐模块里排除禁用项，每类至少选1个，整篇至少换2个模块

**手法层（第3层）：**
④ 查追踪表确定禁用手法（上次每种信息块用了什么手法，这次不能用同一种）
⑤ 按比赛场景查"每种场景推荐手法表"，为每个信息块选一种不同的手法

**写稿时注意：** 三步互不耦合。风格选叙事流不代表模块必须选背景叙事钩子，模块选单点爆破式不代表呈现手法必须选数据流式。独立排列组合才能拉开差异化。选定后自然融入叙事，不要出现"这里开始写数据板块"的生硬感。

**第三步：写正文（按选定类型走）**

**类型A（钩子版）结构：**
1. **个人反应开头** — "说实话第一次翻数据愣了下" / "翻到这条记录的时候我停了一下"（不要"先给你们一组数字"）
2. **数据展开** — 核心数据自然引出，不单独开板块
3. **球队分析** — 两边轮流聊，不要"先说A再说B"的工整结构
4. **首发预测+战术观察** — 给出具体球员名字和阵型分析
5. **未来赛程** — 两队后续赛程对比，增加信息量
6. **赔率/外媒** — 自然叙述（"外媒那边也挺一致..."而非列表格式）
7. **数据包CTA + 自我介绍 + 合规声明**（3段式收尾，见Step 5模板A）

**类型B（增强标准版）结构：**
1. **钩子开头** — 数据反差/背景叙事/场景钩子（按风格库选型）
2. **正文** — 按选定风格（叙事流/对决体/数据先导式等）走，参考写作池模块组合
3. **增强版三要素**（必做）：外媒预测2-5个 + 大小球分析 + 角球数据
4. **赛博数据分析** — 泊松数据/概率/个人猜测
5. **结尾** — 自由组合（互动提问/图一乐/关注引导，见Step 5模板B/C）

**第四步：写完后过一遍——各类型内容完整性确认 + 模块追踪记录**

正文完成后，回到写作池模块追踪表，记录本次实际使用的模块（②开头钩子~⑧结尾收法各用了哪个）。下一篇写稿时对照查。

**完整性确认：**

| 检查项 | 类型A（钩子版） | 类型B（标准版） |
|-------|:-------------:|:--------------:|
| 非数据内容（阵容/赛程/战术/碎碎念）占~30% | ✅ 必须有 | 不强制 |
| 增强版三要素（外媒+大小球+角球） | 不强制（数据包里有） | ✅ 必须有 |
| 数据包 | ✅ 附在文末或单独文件 | ❌ 不需要 |
| 全文650-900字 | ✅ 目标字数 | 500-800字 |

### Step 3：降AI率（朱雀检测，目标50%+人工特征）

**❗ 2026-05-21 更新：DetectRL（NeurIPS 2024）论文验证的降AI策略**

一篇来自NeurIPS 2024的基准论文（DetectRL）系统评测了各类检测器在真实场景下的表现，以下发现直接验证并补充了我们的降AI策略：

**论文核心发现1：对抗扰动（Perturbation）是目前最狠的降检测手段**
- 字符级/词级扰动让零样本检测器AUROC暴跌至38%（≈抛硬币）
- ✅ **已验证有效的手段**：拼写错误、标点搞乱、不完整句、断句打散，都属于扰动类攻击
- ✅ **持续做**：每篇保留3-5处自然噪声（不要系统化注入，随机分布）

**论文核心发现2：短文本比长文本更难检测**
- 60-80词是零样本检测器的"盲区"，文本越长检测器越容易判断
- ✅ 解释了为什么我们600字左右的微头条/短文反而过关率高于850字长文
- ✅ **策略调整**：重点文章控制在600-750字，必要时宁砍不注水。超过800字反而受伤

**论文核心发现3：Paraphrase（改写）能让AUROC掉15%**
- 改写（包括回译、LLM润色、同义词替换）能显著打乱统计特征
- ✅ 我们的 humanizer-zh 去AI连接词 + chinese-level-ai-bypass 多级混合，本质上就是高级paraphrase
- ✅ **持续做**：写完初稿后必须经过改写式降AI，不做"裸发"

**论文核心发现4：Data Mixing（混合写作等级）是论文级策略**
- 把不同模型的输出拼在一起，检测器AUROC下降18%
- ✅ **已验证有效**：chinese-level-ai-bypass Level 4（外国人语感）+ Level 3（小学生句式）+ Level 6（方言/中式英语）混合重写，正好踩中data mixing路线
- ✅ **论文背书确认**：这条路方向正确，继续走

**论文核心发现5："太干净=AI"不是直觉，是论文级结论**
- 对真实人类写作做paraphrase/润色后，误判率反而上升——说明"工整流利"本身就是AI特征
- ✅ 与我们的"整体感>特征打钩"原则完全一致
- ✅ **写稿原则**：保留毛边感（跑题比喻、口语重复、不完整思考），比精心打磨更重要

**论文核心发现6：Claude 生成文本天生最难检测**
- 所有检测器在Claude输出上的AUROC普遍比GPT-3.5低10-20个点
- 提示：不同模型的"统计指纹"差异明显，多模型混合可能有天然优势

**三条新行动指南（基于DetectRL）：**
1. **目标字数下调**：正文控制在600-750字，超过800字时删冗余（非数据内容优先保留）
2. **"短+乱"原则**：短文（600字档）+ 自然噪声（拼写/断句/跑题）= 最难被检测的组合
3. **每次改写手法随机轮换**：不要每次都humanizer-zh → bypass Level 4+3+6同一套流程。这次用paraphrase主导，下次用扰动主导，再下次用混合等级，让检测器无法形成"固定套路"

**禁用句式：**
- ❌ "先聊点背景" / "先说说XX" / "再来说说XX"
- ❌ "不过赛前有点麻烦" / "但值得注意的是"
- ❌ "所以这场两边都有XX"（"所以"总结句式）
- ❌ "突然想到XX" （"突然想到"是AI标志）
- ❌ "几个我觉得值得关注的点"（列举框架）
- ❌ 每段"主题句+展开论证"的工整结构
- ❌ "不过""然而""此外"等过渡词堆砌
- ❌ 完整平衡的长句（看起来太"完美"）
- ❌ **整齐的外媒列表格式**（"- USA Today：2比0"）→ 改成叙述式（"外媒那边也挺一致，USA Today给2比0"）
- ❌ **显性分段标记**（"数据方面""大小球分析""阵容方面"）→ 自然过渡

**做：**
- ✅ 短句分段，每段1-3句
- ✅ 口语词点缀（"凑合""邪门""说实在的""不好说"）

### Step 5：发布后互动（必做，这是突破展现量瓶颈的关键）

**为什么必须做：** 报告数据确认——5月13日大盘11,924展现的当天评论量是6条。0-2条评论的文章展现全卡在2,000-3,000档。互动是触发二级推荐池的唯一杠杆。

**三阶段互动策略（匹配推荐漏斗）：**

**阶段一·冷启动期（发布后0-1小时）— 标题点击率已由正文决定，不要乱动**
- 此阶段算法在看点击率，少则得，不需要干预
- 唯一要做的：确认文章已正常发布、审核通过

**阶段二·小流量测试期（发布后1-4小时）— 互动率决定生死**
1. **发布后30分钟内：自己评论并置顶** — 用作者账号写一条带争议性的话题（例："这场我看好下盘，大家怎么看？" / "曼城补赛冲冠，但我偏觉得水晶宫能搅局——你们觉得呢？"）
   - ⚠️ **评论区也在审核范围内**：置顶评论不能写"点赞+关注+转发"三连诱导，会被判违规引导。用争议型问题是安全的
2. **发布后1小时：检查评论量** — 如果自然评论<2条，自己再发一条不同的争议评论顶上去（不要跟置顶重复角度）。首条评论的角度决定了讨论的方向
3. **任何新评论，30分钟内回复** — 每条都回，哪怕只回"确实"或"哈哈"。算法会因为回复率高而加推。回复长度不超过原评论。

**阶段三·放大期（发布后4小时+）— 维持互动热度**
- 持续关注评论区，新评论立即回复
- 遇到打脸预测不自辩，直接认（"哈哈确实看走眼了"）
- 不删负面评论——争议本身就是流量
- 如果当天互动特别高（10条+），第二天可以再回复一轮，激活二次推荐

**争议型问题要尖锐：** 不用"你觉得谁赢面大"（太温和）。用让读者站队的问题（"XX真能救这支球队？评论区站队"）。

1. **翻视频文案矩阵.md** — 找到当前文章风格对应的口播叙事线（数据先导式→①抛数字②对比展开③主队④客队⑤变量⑥外媒⑦倾向⑧互动）

## 五、矩阵文章（赛后复盘）

比赛结束后，对比赛前分析与实际赛果，写复盘回顾。

### 风格一：FotMob式（时间线战报+复盘，80%/20%）

**特点：** 按比赛时间线推进，把复盘点嵌在关键事件里，读起来是战报，但穿插"赛前分析说过"。

**结构：**
1. 电头/一句话结果
2. 按时间线讲故事（赖斯封堵→刘易斯-斯凯利→萨卡进球→加布救险→哲凯赖什打飞）
3. 关键球员一段（如格列兹曼告别、萨卡回归）
4. 数据验证（xG对比，复盘赛前分析准不准）
5. 结尾统一格式

**写作注意：**
- 不要分割成"赛前说了什么vs实际发生了什么"两块，而是融进去
- 复盘点自然带出，不打断叙事节奏
- 数据对比用xG等客观指标

### 风格二：ESPN式（多板块深度专题）

**特点：** 新闻电头开头（"伦敦——阿森纳..."），多视角分板块，每个板块一个主题，赢家输家各有独立篇幅，带背景故事和深度分析。

**结构：**
1. **电头开头** — 地点+结果一句话（"伦敦——阿森纳时隔20年..."）
2. **赢家板块** — 主队历史意义、教练去留、关键球员故事
3. **输家板块** — 客队失利原因、教练决策、核心球员告别
4. **球星特写** — 单独一段写一个关键球员（如萨卡、格列兹曼）
5. **战术/数据段** — 防线表现、关键数据
6. **展望** — 下一场/决赛前瞻

**写作注意：**
- 每个板块200-300字，独立成段
- 视角平衡，不偏袒一方
- 带背景深度（历史战绩、合同情况、纪录数据）
- 语言偏正式体育媒体风格

### 风格三：SI式（话题深度调查 — 参考用）

**特点：** 围绕一个话题（转会/争议/球员关系）做多方调查，问题驱动结构，从头到尾是"一个悬念的展开"。

**适用场景：** 场外话题、转会传闻、球员争议、深度人物特写。目前账号做的不多，作为风格储备。

**可借鉴的点：**
- **"先立靶再反转"开头** — "都说XX要走了，但俱乐部根本没考虑卖"
- **问题式小标题** — 用设问做分段标题（Why / Could / How），比普通标题更有钩子
- **开放式结尾** — 不强行总结定论，留个思考给读者
- 这些写法不仅限于转会话题，赛前分析也能用（"为什么这场阿森纳能赢？" 然后分点展开）

### 三种风格怎么选

| 场景 | 推荐风格 |
|------|---------|
| 快速出稿，偏战报 | FotMob式 |
| 重要比赛，想深度分析 | ESPN式 |
| 时间紧张 | FotMob式，30分钟搞定 |
| 周末/大战后专题 | ESPN式，可配图分板块 |
| 场外话题/转会/人物特写（储备） | SI式（参考问题式小标题+开放式结尾） |
