# 阿飞·赛博聊球 — 文章生产技能

## 项目定位

头条号"阿飞·赛博聊球"，人设是用数据拆解比赛的分析师，不是荐彩大神。口头禅："数据不会说谎，但人会。——阿飞·赛博聊球"。文章目标：单篇1000-2500字，50%+人工特征通过AI检测。

## 触发条件

用户提供比赛的中国竞彩网数据（含积分榜、近况、射手、伤停），要求写赛前前瞻文章。

---

## 灵活文章框架（非固定模板，是元素池）

以下为文章应包含的核心元素，但顺序和呈现方式每篇不同。禁止每篇用相同骨架。

**必备元素（每篇必含）：**
- 标题：日期+周X+队名VS队名+钩子，25-30字
- #话题标签# 3个以内
- 时间·球场·赛事轮次
- "数据不会说谎，但人会。——阿飞·赛博聊球"
- 核心数据钩子（开篇抓人）
- 主队近况+伤停+核心球员
- 客队近况+伤停+核心球员
- 外媒分析预测（3-5家实测搜索，叙述式非列表式）
- 阿飞解码/看法（模型数据+个人判断+叙事收束）——模型输出融入段落不单独开板块，详见"数学模型分析规范"
- 比分预测（四源综合）——外媒预测 + SPF赔率隐含概率 + 让球盘口走势 + 泊松/蒙特卡洛模型概率。格式：个人猜测比分方向：X-X 或 X-X（一句话总结媒体分析+sp水位+盘口+模型）。四源一致时给结论，不一致时呈现分歧本身就是价值
- "难说。真说不好。"
- 互动提问
- 两行固定免责："图片及数据来源网络侵删。" / "纯个人图一乐的看法，投资需谨慎。"
- 数据来源列表

**写作风格禁忌：**
- 不用【】做分段标记——用自然段落过渡、口语化转折
- 不用"首先/其次/最后"或"值得注意的是/综上所述"
- 不列整齐的外媒列表，改成叙述式
- 信息密度不均匀——有的段落800字有的1行，模拟真人写作节奏

## 写作池三层轮换机制

每篇从三个独立层各选元素，三层排列组合确保没有两篇雷同。

### 第一层：风格骨架（7种，不连用同一种）
分板块式 / 对决体 / 数据先导式 / 战术拆解式 / 叙事流 / 清单体 / 微头条
选型原则：比赛有什么故事→上一篇用了什么（不连用）→出稿时间。

### 第二层：写作模块（8大模块，每篇至少换2个）

| 模块 | 可选写法数 | 说明 |
|------|-----------|------|
| 开头钩子 | 6种 | 背景叙事/争议/数据反差/历史/结论前置/场景 |
| 数据板块 | 3种 | 三层递进（推荐）/对比式/单点爆破 |
| 背景叙事 | 3种 | 赛季叙事线/搅局者/首回合呼应 |
| 球星板块 | 3种 | 故事型（推荐）/数据型/对位型 |
| 伤病板块 | 2种 | 清单式/影响评估式 |
| 战意速查 | 3档 | 全力争胜(1.2-1.5)/轮换放水(0.6-0.8)/默契平局(0.9-1.0) |
| 预测板块 | 3种 | 概率比分理由/倾向依据/争议式 |
| 结尾板块 | 4种 | 数据静默/争议提问/意象收束/合规互动 |

### 第三层：呈现手法（8种信息块各4-5种写法，至少换3个）
同一信息块（如近期战绩、外媒分析）在不同文章用不同写法。例如外媒可用一致性呈现/分歧聚焦/层层递进/列举反打。同一手法不连用两篇。

### 第四层：叙事深度素材包（6种，每篇选1-2个嵌入）
A恩怨叙事 / B戏剧性类比 / C人文深度 / D宿命文学 / E足球热梗 / F战术演变
嵌入方式：融进正文段落，不单独开板块。一重一轻搭配（如恩怨+热梗，宿命+戏剧性）。同素材包不连用。

### 叙事池维度（10个，每篇选2-3个嵌入正文）
恩怨情仇 / 教练博弈 / 战术要素 / 预测首发阵容 / 球员转会 / 双方战意 / 射手介绍 / 热梗 / 裁判因素 / 杯赛欧战影响
同维度不连续两篇用同一个角度。

### 比赛类型速查（模块组合推荐）
- 争冠关键战 → 背景叙事钩子+赛季叙事线+搅局者+三层递进数据+故事型球星+概率比分预测+争议提问收
- 强弱分明 → 数据反差钩子+首回合呼应+对比数据+数据型球星+倾向依据预测+数据静默收。弱队保级则必加战意速查
- 强强对话 → 场景钩子+赛季叙事线+对比数据+对位型球星+争议式预测+意象收束收
- 杯赛小组末轮(或类似生死局) → 数据反差钩子+战意速查(必加，放开头附近)+出线形势+三层递进数据+倾向依据预测+争议提问收

## 生产工作流

1. 接收竞彩网数据 → 读数据找出叙事钩子
2. WebSearch搜外媒（至少3家不同国家来源，实测搜索后引用）
3. WebSearch搜SPF赔率和让球盘（中国竞彩网或500彩票网等），获取最新水位+盘口数据
4. 数据核查：源数据内部一致性检查（赛季总成绩vs积分榜可能冲突）、排名描述必须可验证、不确定的数字宁可不写、历史纪录必须联网验证
5. 运行数学模型分析（详见"数学模型分析规范"章节）：xG估算→泊松分布→蒙特卡洛10万次→敏感性分析。模型输出作为阿飞解码/比分预测的素材之一
6. 选风格→选模块→选呈现手法→选素材包→选叙事维度（五层各独立轮换）
7. 写初稿（1000-2500字，以信息传递为主）
8. 降AI两步流水线
9. 数据+合规终检

## 🚨 强制核查与风险清单（生产前必检，不可跳过）

以下三项为每篇文章动笔前必须完成的硬性检查，漏一项=废稿。

### 一、数据真实性核查

**必须联网验证的信息（不得依赖训练数据中的"知识"）：**
- 球员姓名、号码、位置、国籍 → 竞彩网或官方渠道核实
- 球队首发阵容、主教练姓名 → 必须搜索最新赛前发布会/球队官网确认
- 对战历史纪录（交锋比分、时间、赛事） → 至少两个独立来源交叉验证
- 球员转会状态、是否仍在队中 → 必须搜索最新转会新闻
- 伤停名单 → 必须搜索赛前伤情更新，标注"截至发稿时"
- 球场名称、开球时间 → 必须与竞彩网/官方赛程核对

**核查方式**：每写到一个具体名字或数字，脑中问"这个信息我能找到来源吗？"答"否"就不写。

### 二、今日头条平台风险红线（禁止博彩相关）

以下内容**绝对禁止**出现在文章中，触发任一即可能被下架/封号：

| 禁止内容 | 说明 |
|----------|------|
| 博彩平台名称 | "竞彩网"可提（官彩），但"威廉希尔""bet365""皇冠"等境外博彩平台名禁止 |
| 投注引导用语 | "推荐""稳胆""上车""跟单""重注""梭哈""all in"等 |
| 晒投注单/中奖截图 | 任何形式的实际投注凭证 |
| 赔率数字诱导 | 不得出现"赔率X.XX值得入手""这个赔率很香"等暗示购买的语言 |
| 付费推荐导流 | "加V私推""付费群""包月推荐"等 |
| 诱导互动购彩 | 评论区互动不得暗示"买了吗""上车没""收米了吗" |

### 三、合规软化词（必须使用）

- "个人猜测比分方向" — 不可省略"猜测"二字
- "纯个人图一乐的看法，投资需谨慎。" — 文末固定免责，不可省略
- "难说。真说不好。" — 比分预测后必加
- 让球盘描述用法："竞彩让球盘主让一球"，不可用"亚盘"

**速记口诀**：名字必核实、博彩不沾边、猜测挂嘴边、免责收尾必写全。

## 标题铁律

核心公式：**数据+悬念，不是数据+答案。** 标题留未回答的问题。

8种标题公式（按比赛情况选）：
- 数据反差+情绪词（有反差数据时）
- 人物冲突+悬念（有焦点球星时）
- 反常识提问（有冷门可能时）
- 时间紧迫+必看（赛前2小时内）
- 争议站队（需要评论区互动时）
- 叙事判断句式（数据一边倒时）
- 大格局叙事（争冠/保级关键轮）
- 二段/三段式钩子式（日常主力，25-30字占爆款67%）

自检：看完标题还想知道后面吗？答"否"就重写。

## 外媒整合规范

必须实测搜索后引用，3-5家不同国家来源。常用库：Sports Illustrated、Yahoo Sports/AP、Flashscore、Sports Mole、Infobae Argentina、AS España、Bolavip、DraftKings、Sid Lowe/Guardian、BBC Sport。

**格式铁律**：叙述式不用列表式。不要"来源A：2比1 / 来源B：1比0"——改成"Sports Illustrated给了1比0，但跟了一句绝不能被低估。DraftKings那边倒是看好巴列卡诺。"呈现分歧比统一结论更有价值。

## 数学模型分析规范

### 工具文件
`poisson_monte_carlo.py`（5步完整流程，含泊松+蒙特卡洛+敏感性分析）、`poisson_analysis.py`（可复用函数库，含泊松概率/胜平负/总进球/最可能比分/期望进球全套函数）。

### 运行时机
生产工作流第5步，在数据核查之后、风格选型之前。模型输出作为阿飞解码/比分预测的素材之一，不是单独板块。

### 五步流程

**Step 1: xG估算** — 基于球队近期攻防数据估算主客队期望进球。淘汰赛保守下调（-0.15~0.30），主力伤缺再扣除。xG取值不是纯数学，包含主观判断。参考依据：竞彩网提供的近期场均进球+最近6场趋势。

**Step 2: 泊松分布** — `poisson.pmf(k, λ)` 算双方独立进球概率，相乘得比分矩阵。输出：最可能比分Top10 + 胜平负概率汇总 + 总进球分布。

**Step 3: 蒙特卡洛模拟** — `np.random.poisson(λ, 100000)` 采样10万次。用随机采样交叉验证泊松结果，额外输出大小球概率。蒙特卡洛与泊松结论一致时可信度更高，不一致时如实提。

**Step 4: 敏感性分析** — 变化xG±0.2，看胜平负概率是否稳定。浮动超过5个百分点→预测可信度打折扣→文章中如实说"模型也说不死"。

**Step 5: 晋级概率**（仅两回合淘汰赛） — 结合首回合比分计算直接晋级/加时/出局概率。

### 文章呈现铁律

1. **不单独开板块**：模型结果融入阿飞解码段落，不要开"数学模型分析"分节。就自然带一句"拿数据跑了一下模型"之类的话。

2. **口语化包装**：永远别说"泊松分布计算显示主胜概率63.2%"。改成"拿数据跑了一下，主胜大概六成多一点"。数字可保留但语气像人在说话。

3. **选择性呈现**：五步输出不是每步都写。通常只取2-3个最相关的发现（如最可能比分+胜平负概率+大小球倾向），其余不写。

4. **自嘲型引用**：模型不是神。引用模型后加自嘲："当然，模型算不出更衣室里的情绪"、"球场上什么事都能发生，概率只是概率"、"说实话我写这段的时候也在想——这种比赛模型能算出来才有鬼"。

5. **不均匀注入**：模型数据只出现在阿飞解码1-2个自然段，其他段落零模型痕迹。不要每段都cue模型。

6. **分歧即价值**：模型结论和外媒/SPF不一致的时候反而是好素材——"模型觉得主胜概率高，但sp那边平局赔得特别低，市场明显在防平。那就得想想了。"

### SPF水位+盘口数据获取

工作流第3步搜竞彩网或500彩票网获取：
- **SPF赔率**：胜/平/负对应的sp值（隐含概率 ≈ 1÷sp，但含庄家抽水，需去水处理）
- **让球盘**：让球方向和盘口深度（如"主让一球"或"客受半球"）
- **水位变化**：初盘→即时盘的调整方向（升水=庄家不看好，降水=庄家看好）

这些数据与模型输出一起融入阿飞解码，对比sp隐含概率与模型概率的差异。差异大的方向天然有叙事张力。

### 比分预测四源合成格式

比分预测段落总结四个独立来源：

| 来源 | 内容 | 呈现方式 |
|------|------|----------|
| 外媒预测 | 3-5家比分/胜负判断 | 叙述式总结，有分歧就呈现分歧 |
| SPF水位 | sp值隐含概率+变化方向 | "SPF那边给主胜X.XX，市场..." |
| 让球盘口 | 盘口深度+水位走势 | "盘口主让一球，初盘到即时盘X→X..." |
| 数学模型 | 泊松/蒙特卡洛概率 | "模型跑了一下，最可能的比分是..." |

四源一致的场合给结论，不一致的场合呈现分歧本身——"四个来源各说各话，这种情况也挺有意思的"。

### 代码依赖
需要 `numpy` 和 `scipy.stats`。运行前修改代码内的 `xG_home` 和 `xG_away` 变量。

## 降AI两步流水线

### Step A: humanizer-zh（去AI味）
去掉：过度连接词、四字成语连用（两个以上）、模板句式（"值得注意的是""综上所述"）、被动语态、过度解释、均匀句长。

### Step B: chinese-level-ai-bypass Level 4+3+6混合（加人味）

| Level | 特征 | 嵌入位置 |
|-------|------|---------|
| 4（外国人学中文） | 短句、犹豫停顿、过度解释简单概念、怎么说呢/就是…那种… | 叙事转折处、数据解读处 |
| 3（小学生水平） | 很…很…、重复短句、然后…然后…、直白不拐弯 | 情绪高潮处、下判断时 |
| 6（方言区中文） | 整/搁/犯不着/闹的/靠/呗/嘛/嘞、直接粗粝 | 吐槽、反问、收束时 |

每篇必须包含（至少各1处，不均匀分布）：
- 脏话/粗口（"卧槽""他妈的""靠"）
- 跑题比喻或联想
- 不完整句（句号或省略号突然打断）
- 同一词重复（"没来过。也没来过。"）
- 视角滑动（第一人称突然插入，如"说实话我写这段的时候也在想"）

**核心纪律**：人工痕迹不均匀分布——大部分段落正常写，只在3-5个点位各插1句。均匀注入=AI特征。

## 彩民文化嵌入（公众号/头条号必备）

文章中自然嵌入经典彩民语录，融入正文不单独开板块。可选语录库：

**玄学定律类：**
- "小鸟一叫，生死难料。"（雷速体育社区黑话，指比分界面弹出提示音时常有反转剧情）
- "球是圆的，什么结果都可能发生。"
- "强强对话多平局，榜首榜尾出妖人。"
- "欧冠无弱旅，英超无冷门。"（反讽用法）

**足彩调侃类：**
- "买球买冷门，别墅靠海边。"
- "强队看盘口，弱队买进球。"
- "生死看淡，不服就干。"
- "足彩本质是小赌怡情，重注伤身。"

**巨星定心丸类：**
- "要想生活过得去，还得相信C罗去。"
- "相信梅西，绝不暴毙。"

用法：在阿飞解码或比分预测段落中，自然嵌入1-2句作为调侃收束，例如"球是圆的——这话虽然老套但今晚确实适用"。不要每篇都用同一句，轮换。

## 数据铁律与合规红线

**数据绝对禁止：**
- 编造任何数据、外媒观点、历史纪录、球员信息
- 依赖训练数据中的"知识"（姆巴佩是否转会、某队上次夺冠年份等）——必须联网验证
- 排名描述加无法验证的轮次限定词（"前10轮一直排第X"除非有逐轮数据）

**合规绝对禁止：**
- "稳胆""必收""百分百""包赢"等绝对判断词
- 赌博平台导流、晒投注单、付费推荐
- 使用"亚盘"描述竞彩让球盘（正确说法"让球盘主让一球"）

**必加元素（漏了就违规）：**
- 比分预测必加"个人猜测比分方向"软化词 + "难说。真说不好。"
- 文末两行固定免责，不可省略
- 互动提问不可含"点赞+评论"双条件（触发诱导互动判定）
- 源数据内部冲突时向用户确认用哪个版本

## 反模式（强制禁止）

- 每篇用相同的结构骨架（检测器秒识别）
- 整齐的外媒列表格式
- 四字成语连用两个以上、对仗排比句
- 标题给答案不给悬念
- 漏掉免责声明或"个人猜测"软化词
- 信息密度过高——太完整=AI特征
- 人工痕迹均匀注入每段——系统性注入本身就是模式
- 依赖训练数据中的历史事实——必须联网验证
- 用【】做显性分段标记——每篇固定位置出现同样标记是最大AI破绽
- 🚨 跳过强制核查清单直接动笔——球员/教练/对战/首发信息未核实就写=必翻车
- 🚨 出现博彩平台名称、投注引导词、赔率诱导语——触发平台审核红线
- 🚨 编造外媒观点或数据——即使看起来"合理"也不允许，必须是真实搜索到的内容
