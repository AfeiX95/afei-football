# 阿飞·赛博聊球 — 风格库

选风格之前先想三件事：
1. **比赛性质** — 淘汰赛还是联赛？豪门还是中下游？
2. **核心故事** — 这场最好聊的点是什么？（一个数据？一个对位？一段历史？）
3. **出稿速度** — 热点要抢还是可以慢慢写？


## 风格一：分板块式
**一句话**：该聊的都聊到，规整不出错。

**适用场景**：中立观众为主、信息量大、什么维度都想覆盖到。
**慎用**：连着用两场以上，读者会腻。

**结构模板**：
```
开场 → 首回合/近况回顾 → 主队 → 客队 → 赛博数据分析 → 唠几个点 → 结尾
```

**代表作**：弗赖堡vs布拉加（初稿）
**全文长度**：800-1200字


## 风格二：对决体
**一句话**：矛对盾，风格对撞，谁克谁。

**适用场景**：两支球队风格迥异（攻击vs防反、控球vs绞杀）、势均力敌。
**慎用**：强弱太分明的比赛用不起来。

**结构模板**：
```
开场亮人设 + 悬念 → "先说一下A"（A的强势+隐忧）→ "再说B"（B的强势+隐忧）
→ "两个数字/两种风格的对话"（把矛盾顶在一起）→ 赛博数据分析 → 结尾
```

**代表作**：维拉vs诺丁汉
**全文长度**：800-1000字


## 风格三：数据先导式
**一句话**：一个离谱数字撑全场。

**适用场景**：H2H碾压、某项数据极端突出（主场10连胜、10场不败、某射手爆种）。
**启动条件**：先翻数据，找到一个"卧槽"级别的数字。

**结构模板**：
```
用那个数字开场 → "先说一个数字"（展开） → 主队情况 → 客队情况
→ 这个数字为什么不正常/为什么合理 → 赛博数据分析 → 结尾
```

**代表作**：多特vs法兰克福（主场9胜1负）
**全文长度**：600-900字（数据撑场，不需要写太满）


## 风格四：战术拆解式
**一句话**：冷冰冰的战术板，像教练在赛前更衣室讲。

**适用场景**：淘汰赛次回合、战术含量高（阵型克制、关键对位决定走向）。
**慎用**：普通联赛用这个显得用力过猛。

**结构模板**：
```
晋级形势图（表格/路线）→ 预计首发阵型 → 关键对位（3个左右）
→ 双方攻防命门 → 赛博数据分析 → 一句话收尾
```

**代表作**：无（待创作）
**全文长度**：700-1000字
**语言特点**：少口语、多术语、不闲聊


## 风格五：叙事流
**一句话**：不止聊球，聊人聊历史聊故事。

**适用场景**：重大对决（欧冠决赛、国家德比）、有丰富历史故事可挖。
**启动条件**：两个队之间有故事——恩怨、数据、旧将、传承。
**慎用**：中下游保级战？没必要。

**结构模板**：
```
场景/画面开头（把读者拉进去）→ 旧事重提（历史铺垫）
→ 这场为什么不同 → 胜负手分析 → 不长的收尾
```

**代表作**：无（待创作）
**全文长度**：1000-1500字
**语言差异**：更文学化，可以有比喻、场景描写、倒叙


## 风格六：清单体
**一句话**：直给，不废话，看完就懂。

**适用场景**：普通联赛快速出稿、读者只需要重点不需要全貌。
**最佳搭档**：周一/周五多发，读者刷手机时看。

**结构模板**：
```
开场一句总起 → 看点1/2/3/4/5（每段3-4行）→ 赛博数据分析 → 一句结尾
看点举例：关键球员、阵型博弈、X因素、历史玄学、场外信息
```

**代表作**：无（待创作）
**全文长度**：500-700字
**语言差异**：段落更短，每段只讲一个事


## 风格七：微头条
**一句话**：热点话题，短平快，占住流量就走。

**适用场景**：平台话题活动、赛后热点发酵、不需要深度的场次。
**禁忌**：不要塞分析结构进去，不是每件事都需要赛博数据分析。

**结构模板**：
```
标签（官方话题放第一）→ 一句话切入 → 2-3段关键信息 → 抛问题给评论区
```

**代表作**：巴黎连续两年进决赛 / 阿森纳冠军梦
**全文长度**：300-500字


## 快速选型表

| 比赛类型 | 推荐风格 | 出稿速度 |
|----------|----------|----------|
| 欧冠/欧联淘汰赛次回合 | 战术拆解式 / 叙事流 | 慢（需准备） |
| 强强对话，联赛争冠 | 对决体 / 叙事流 | 中 |
| 强弱分明 | 数据先导式 / 清单体 | 快 |
| 普通联赛 | 清单体 / 分板块式 | 中 |
| 热点话题/平台活动 | 微头条 | 最快 |
| 重大决赛 | 叙事流 | 最慢（要打磨） |

优先顺序：
1. 先看比赛**有什么故事可讲** → 定风格
2. 再看**出稿时间** → 砍内容还是加内容
3. 最后看**上一篇用了什么风格** → 尽量不连用同一种
